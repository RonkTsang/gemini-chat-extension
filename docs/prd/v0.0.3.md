实现“消息内容快速引用”功能

## 背景：
在与 Gemini AI 的对话中，AI 可能会给出较多的文字内容和信息，此时用户会想对其中某段内容进行二次追问，此功能提供一个快捷的选中内容并追问的快捷方式

## 具体逻辑：
1. 监听模型回答的区域中（所有的`<message-content />`），用户的文本选中事件
2. 当用户选中某端文本，在选中结束后，鼠标上方出现一个优雅的可交互 Tooltips 按钮（追问按钮），内容为 “Ask Gemini”，位置默认为选中结束时刻的鼠标上方，需要做边缘空间检测
3. 当用户点击“追问按钮”，追问按钮消失，向用户输入框插入一段隐藏的文本，并focus在输入框，可参考以下：（输入框DOM结构见最后补充）

```html
<p style="display: none; user-select: none; pointer-events: none;" id="gemini-fake-content">
  Referring to this in particular: {用户选择的内容}
  <br/>
</p>

```
4. 同时在输入框上方插入 UI（称为：“引用状态UI”），已表明当前正在引用 {用户选中的} 内容，这部分组成为
  - 引用的标识icon
  - 内容原文
  - 右侧关闭引用小按钮：点击之后移除“引用状态UI”，并删除隐藏的文本
5. 若用户未点击“追问按钮”，点击页面其他位置，“追问按钮” 消失

## Dom 结构补充
1. 模式回答会以以下DOM结构：注意，`chat-window` 下可能会存在多个 `model-response`，随着聊天进行而增加。
```html
<chat-window>
  <model-response>
    <div>
      <response-container>
        <div class="response-container">
          <div class="presented-response-container">
            <div class="response-container-content">
              <div class="response-content">
                <!-- 模型回答 -->
                <message-content />
              </div>
            </div>
          </div>
        </div>
      </response-container>
    </div>
  </model-response>
</chat-window>
```

2. 输入框DOM结构
```html
<input-container>
  <!-- 略过多个层级 -->
  <xxx> 
    <rich-textarea>

      {“引用状态UI”}

      <div class="ql-editor ql-blank textarea new-input-ui" data-gramm="false" contenteditable="true" role="textbox" aria-multiline="true" aria-label="Enter a prompt here" data-placeholder="Ask Gemini">
        <!-- 隐藏的文本所插入的位置 -->
        {隐藏的文本}
      </div>
    </rich-textarea>
  </xxx>
</input-container>
```
