# 首页 UI 优化建议

## 目标

把首页从关键词堆叠的文档入口，改成能在首屏说明产品价值的产品页：清晰、安静、有真实产品感。

SEO 不需要依赖可见标题堆词。保留现有 `<title>`、description、JSON-LD 与功能页关键词覆盖；将视觉 H1 改为面向用户的价值主张。

## 当前问题

- H1 直接使用 SEO 标题，且被限制在窄宽度内，桌面和移动端均严重断行。
- Hero、介绍、价值卡和功能卡反复说明同一批功能，阅读没有推进。
- 过多圆角、描边、阴影和半透明卡片，使所有内容同等抢眼。
- 移动端连续堆叠七张以上卡片，页面过长；浅色截图在白色卡片里缺乏边界。
- 目前字体字重偏重、正文全局字距偏松，不够克制。

## 内容结构

1. **Hero**：一句价值主张、一句说明、安装 CTA、真实 Gemini 使用截图。
2. **能力概览**：三句无卡片短文，概括导航、追问、自动化。
3. **功能证明**：保留四项功能，使用 2×2 可点击模块；截图是主角，文案简短。
4. **信任信息**：一条本地处理承诺，一个反馈入口。
5. **收尾**：只保留一个安装 CTA；文档入口降为次级文本链接。

## Hero 文案

```text
Gemini Power Kit for Chrome

Make Gemini easier to work in.

Navigate long conversations, follow up from selected text,
and turn repeat tasks into reusable flows.

[Add to Chrome]  [See how it works]
```

中文：

```text
Gemini Power Kit · Chrome 扩展

让 Gemini 更顺手。

在长对话中快速定位，选中文本即可追问，
把重复任务整理为可复用的工作流。

[安装到 Chrome]  [查看功能]
```

## 视觉规则

- Hero 使用真实产品截图，应用图标只保留在导航栏。
- 背景用接近 `#F5F5F7` 的暖灰；大面积留白取代 Hero 大卡片。
- 仅截图、功能模块和最终 CTA 使用表面容器；不要给每个段落套卡片。
- 使用系统字体优先：标题桌面 `64px / 1.04 / -0.035em`，移动 `42px / 1.08`；正文 `17px / 1.55 / 0`。
- 圆角分级：大模块 `28px`、图片 `20px`、按钮胶囊形。浅色截图需有 `1px` 中性描边与轻阴影。
- 主按钮只保留“Add to Chrome”；文档和 GitHub 使用低强调链接。

## 交互规则

- 按钮按下立即缩放至 `0.98`。
- 卡片 hover 仅使用 160–200ms 的边框和阴影变化，不使用明显上浮。
- `prefers-reduced-motion` 下仅保留透明度或颜色变化。

## 实施顺序

1. 将页面 SEO 标题与可见 Hero H1 解耦。
2. 重写首页信息结构，删除重复文案和多余 CTA。
3. 建立简化的表面、排版与间距规则。
4. 用桌面和移动端截图复查首屏、图片边界和内容密度。
5. 按相同信息架构同步中文首页。

## Design Taste 复审

### 结构性问题

- `.value-band` 是标准三等分卡片，属于最常见的营销页模板；不应继续保留。
- `showcase-card-wide` 和 `showcase-card-large` 没有跨列或跨行规则，四个功能视觉上仍是完全对称的 2×2 网格。
- 信任区是三张全宽卡片，最终 CTA 又是两张同权重大卡。页面连续出现多组相同容器，信息层级被抹平。
- Hero 的主 CTA 是 “Quick Start”，而安装 CTA 在同级且在页尾再次出现，转化路径不够明确。
- 第一张真实产品截图在首屏之后；Hero 使用图标，无法证明产品实际如何工作。

### 推荐版式

采用左文右图、非对称但可控的产品页面，而不是继续堆叠卡片：

```text
Hero           5/12 价值主张 + 安装按钮 | 7/12 真实 Gemini UI
能力概览       三个短句，以分隔线和留白分组，不使用卡片
功能证明       7/12 Quick Follow-up 主展示 | 5/12 主题与大纲交错展示
信任与反馈     两条横向信息：本地处理 / 提交反馈
收尾           一个安装 CTA；文档链接作为次级操作
```

功能区应有明确的阅读顺序：先展示最易感知的 Quick Follow-up，再展示 Chat Outline、Chain Prompt 和主题。移动端统一折叠为单列，保持截图与文字成对出现。

### 样式改进

- 删除 Hero、价值区和信任区的重复渐变、内阴影与大圆角；容器只用于承载真实截图或唯一的最终 CTA。
- 保留一个低饱和青色作为唯一强调色；紫色仅可作为应用图标本身的品牌细节，不扩散到页面效果。
- 不再全局设置 `body` 字距、`h1` 或 `.card`。为首页建立独立根类和设计 token，避免影响文档页。
- `background-attachment: fixed` 不用于移动端页面背景；它没有信息价值，且可能增加滚动绘制成本。
- 产品图不要统一强制 `16 / 9` 裁剪。不同功能图片放入有中性背景和描边的媒体框，优先完整展示功能界面。
- 将按钮和链接的状态统一：hover 只改变颜色、描边或轻微阴影；active 使用 `scale(0.98)`。移除 `translateY(-6px)` 和大范围暗阴影。

### 动效边界

首页不是仪表盘，不需要循环动画或磁吸按钮。仅在首屏和功能区使用一次性的、渐进 120ms 间隔的 `opacity + translateY` 入场；所有动画只使用 `transform` 和 `opacity`，并在 `prefers-reduced-motion` 下关闭位移。

### 验收标准

- 桌面端 H1 不超过三行，首屏内出现真实产品界面与安装按钮。
- 移动端 H1 不超过三行，首屏没有横向溢出，功能模块只保留单列。
- 页面只保留一个强主 CTA：安装扩展。
- 三张等宽价值卡、三张信任卡、双大 CTA 卡全部移除或合并。
- 任一浅色截图在浅背景下都能一眼看到边界和主要 UI。
